# 常见问题

## 1. 这个项目是不是从零实现 GaussDB 协议？

不是。

本项目定位是 GaussDB SQLAlchemy 方言和适配层。底层连接能力复用华为 `gaussdb` Python DB-API 包，项目重点解决 SQLAlchemy 连接串、方言注册、版本识别、连接参数、事务、ORM 和反射等应用侧接入问题。

## 2. 为什么安装了 gaussdb 仍然报 no pq wrapper available？

这表示 Python 已经能找到 `gaussdb` 包，但没有找到可用的 GaussDB/libpq native client 实现。

Windows 上通常需要：

- 安装 GaussDB 客户端包
- 将客户端 `bin` 目录加入 `PATH`
- 重新打开终端或重启应用进程
- 确认 Python 位数和客户端库位数一致

可以运行：

```powershell
python scripts\check_windows_env.py
```

## 3. 为什么方言默认设置 client_encoding=UTF8？

部分 GaussDB 环境默认 `client_encoding` 为 `SQL_ASCII`。在这种情况下，底层 `gaussdb` DB-API 会把文本字段返回为 `bytes`。

本项目默认传入：

```text
client_encoding=UTF8
```

这样 SQLAlchemy 用户默认拿到 Python `str`。如果确实需要其他编码，可以在连接串中覆盖：

```text
gaussdb+gaussdb://user:password@host:port/postgres?client_encoding=LATIN1
```

## 4. 如何跑真实数据库集成测试？

设置环境变量：

```bash
export GAUSSDB_TEST_URL='gaussdb+gaussdb://user:password@host:port/postgres'
pytest -m integration
```

Windows PowerShell：

```powershell
$env:GAUSSDB_TEST_URL = 'gaussdb+gaussdb://user:password@host:port/postgres'
pytest -m integration
```

集成测试会创建并删除 `codex_*_ut` 前缀的临时表。

## 5. 当前已验证哪些 SQLAlchemy 能力？

当前集成测试覆盖：

- SQLAlchemy Core 基础 DDL、DML、查询
- 事务回滚
- 批量插入
- ORM CRUD
- 元数据反射
- 连接池基础复用

真实环境中，GaussDB 内核版本会保存在 `dialect.gaussdb_server_version_info`；`dialect.server_version_info` 使用保守的 PostgreSQL 兼容版本，避免 SQLAlchemy 误用 PostgreSQL 新版本系统表字段。
